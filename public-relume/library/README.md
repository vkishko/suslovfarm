# Правила библиотеки Webflow / Relume-блоков

Эта папка используется как библиотека блоков, которые можно переносить на рабочие страницы сайта.

## Как добавлять новый блок

1. Экспортируйте блок из Webflow в отдельный HTML-файл.
2. Положите HTML-файл в корень папки `public/library`.
   - Пример: `layout-367.html`
   - Пример: `header-26.html`
   - Пример: `testimonial-51.html`
3. Если Webflow экспортировал отдельные CSS/JS/images для блока, оставьте их в соответствующих папках внутри `public/library`.
4. Не переименовывайте классы Webflow вручную.
5. Не удаляйте `id` вида `w-node-...` — они часто отвечают за сетку и позиционирование.

## Как переносить блок на страницу

При переносе блока из `public/library` на рабочую страницу нужно переносить не только HTML-разметку, но и зависимости блока:

- CSS-классы блока;
- `#w-node-...` правила из CSS;
- изображения;
- видео/иконки;
- Webflow-атрибуты `data-*`;
- структуру wrapper-элементов.

Если перенести только `<section>...</section>` без CSS и `w-node` правил, блок может отображаться неправильно.

## Что важно проверять

После вставки блока на страницу нужно проверить:

1. Не сломалась ли сетка.
2. Совпадает ли фон и цвет текста.
3. Не пропали ли изображения.
4. Нет ли битых локальных ссылок.
5. Работают ли табы, слайдеры, dropdown и другие Webflow-интеракции.

## Рекомендация для будущих блоков

Лучший формат для каждого блока:

```text
public/library/
  block-name.html
```

Если у блока есть отдельные стили, желательно рядом добавить:

```text
public/library/
  block-name.html
  block-name.css
```

Но если Webflow хранит все стили в общем CSS-файле, важно при переносе блока искать и переносить все правила, которые относятся к этому блоку.

## Главное правило

Блок из библиотеки считается полным только вместе с:

- HTML;
- CSS;
- `w-node` правилами;
- ассетами;
- Webflow-атрибутами.
